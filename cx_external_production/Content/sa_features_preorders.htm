Configuring Site Settings: Preorders

The Preorders setting is used to define when you want payment authorization for preorders to be processed.

Who Can See and Edit This Setting

This Help file contains information about every possible site setting, however your ability to see or edit a specific site setting will vary:

  • The ability to see or edit a setting depends on the template used by the currently selected site, the type of user you are (internal Digital River user, or client user), and the user roles you have.
  • Some users can see a setting but not change it, and some users cannot see a setting at all. Some users will be able to both see and edit a setting.

The table below outlines who can see and edit this setting. Find the template used by your currently selected site (you can see the template name at the top of the Configure Site Settings page) and then find your user type. The table will then tell you whether you can see a read-only view of how the setting is configured or whether you can configure the setting yourself. Keep in mind that you will also need the role(s) listed to see or edit the setting.

Note: In the table, "disinherited" means that your site no longer uses the template on which the site was built (meaning the site was built using a template but one or more pages have since been customized so the site no longer inherits the features and functionality of the template).

About the Preorders Setting

The picture below shows the Preorders setting area of site settings. The information below the picture defines these settings and what happens if you set them or how to configure them for your store.

Preorder Authorization:

  • Select When Order is Placed if you want authorization done when the order is placed. This is the same time that authorization would occur for any other non-preorder product.
  • Select When Product is Released if you want authorization done when the preorder product becomes available for sale.
