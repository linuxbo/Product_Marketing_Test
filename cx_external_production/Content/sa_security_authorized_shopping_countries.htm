Configuring Site Settings: Authorized Shopping Countries

The authorized shopping countries setting allows you to define which shoppers can make a purchase on your store based on the billing or shipping address they enter during checkout. This feature is especially useful if you want to deny shoppers from a specific location from purchasing on your store or deny shoppers shipping to a certain location (even if they themselves do not reside in that location) from purchasing from your store.

Who Can View or Edit This Setting

Your ability to view or edit a specific site setting varies as follows:

  • The ability to view or edit a setting depends on the template used by the currently selected site, the type of user you are (internal Digital River user, or client user), and the user roles you have.
  • Some users can view a setting but not change it, and some users cannot see a setting at all. Some users can both view and edit a setting.

The table below outlines who can view or edit this setting. Find the template used by your currently selected site (you can see the template name at the top of the Configure Site Settings page) and then find your user type. Configuring the Authorized Shopping Countries Settings

  1. On the Administration menu, select Site, and click Configure Site Settings. The Configure Site Settings page appears.
  2. Click the Security tab.
  3. In the Authorized Shopping Countries area, in the Billing Countries area, for the Allow Billing From option, select how you want to define purchase restrictions based on the billing address of the shopper:
    • Any Country â€“ Allows shoppers with a billing address from any country to purchase from your store. This option can also be viewed as having no shopping restrictions based on billing address.

    • Any Country Except Specifically Denied Countries â€“ Used to define those countries from which you don't want shoppers to be able to select when they enter their billing information. This prevents shoppers in a certain country (or having billing information tied to a specific country) from making a purchase on your store.

    • Only Specifically Allowed Countries â€“ Used to define those countries you want to allow shoppers to choose from when they enter their billing information. This prevents shoppers in a certain country (or having billing information tied to a specific country) from making a purchase on your store.

  4. Depending on the option you selected for the previous step for authorized billing countries, you will have to either add those countries that you want to deny or allow to the locales supported by your store (if you chose to allow any country to be used in the billing information you can skip this step).
    • Use the Search button to locate and then select the countries you want to deny or allow for the locale.
    • By default, some countries will automatically be added to a deny list. These are the countries for which you cannot do business with based on export control restrictions.
    • Supported locales (all other locales besides the default) will inherit what you set up for the default locale. If you want to configure a supported locale differently from the default, click on that locale tab and then select the Customize for This Locale option and add the countries that you want to deny or allow on that locale.
  5. In the Shipping Countries area, for the Allow Shipping To option, select how you want to define purchase restrictions based on the shipping address entered by the shopper (shopper does not have to reside at shipping address for the purchase to be denied):
    • Any Country â€“ Allows shoppers who enter a shipping address to any country to purchase from your store. This option can also be viewed as having no shopping restrictions based on shipping address.

    • Any Authorized Billing Country â€“ Allows shoppers who enter a shipping address in any country that you have defined as an authorized billing country to place an order on your store.

    • Any Country Except Specifically Denied Countries â€“ Used to define those countries you do not want shoppers to be able to enter for the shipping address for the order. This prevents orders that are shipped to those countries you define from being placed on your store.

    • Only Specifically Allowed Countries â€“ Used to define those countries you want to allow shoppers to enter for the shipping address for the order. This allows orders that are shipped to the countries you define to be placed on your store.

  6. Depending on the option you selected for the previous step for authorized shipping countries, you will have to either add those countries that you want to deny or allow to the locales supported by your store (if you chose to allow any country to be used in the shipping information you can skip this step).
    • Use the Search button to locate and then select the countries you want to deny or allow for the locale.
    • By default, some countries will automatically be added to a deny list. These are the countries for which you cannot ship to based on export control restrictions.
    • Supported locales (all other locales besides the default) will inherit what you set up for the default locale. If you want to configure a supported locale differently from the default, click on that locale tab and then select the Customize for This Locale option and add the countries that you want to deny or allow on that locale.
  7. In the Comments area, enter an explanation for why you have defined the restrictions above. Be sure to provide enough detail so someone else can determine why a specific country may have been denied or allowed (as either billing or shipping) in a particular locale.
  8. Click Save.
