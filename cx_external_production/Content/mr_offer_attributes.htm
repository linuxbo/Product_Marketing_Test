Offer Attributes

The following is a list of the attributes that you will see when you view your list of offers in the Command Console.

Attribute

Definition

ID

The unique ID given to the number when it is first created. The ID is mainly used to identify the offer in reporting but may also be seen in the URL/link when a shopper is redeeming an offer by clicking a link, banner ad, or other element used to trigger the offer.

Name

The name of the offer given to it when the offer was created or last modified. Names are not seen by shoppers on your store. They are only used to identify the offer to you in the Command Console.

Type

The type of offer you selected. See Types of Offers You Can Create for more about the different types you might see listed.

Status

The status of the offer, which determines whether the offer can be seen and used by shoppers on your store. There are five possible statuses:

Draft – Draft offers have been created but not yet deployed on your store. This is the default status for a new offer if you do not deploy it when finishing the wizard.

Design – Design offers are offers that have been created and deployed on your store, but have undeployed changes. Undeployed changes cannot be seen by shoppers on your store.

Deployed – Deployed offers are active on your store and can be seen and redeemed by shoppers.

Expired – Expired offers are offers whose end date has passed. Expired offers cannot be seen or redeemed on your store.

Retired – Retired offers are offers that are not longer available on your store. Once an offer is retired it cannot be made active again, however you can make a copy of the offer and use the copy to create a new offer.

Parent Products

The ID of the Parent products for the offer. This only appears when the offer is a bundle, cross-sell, or up-sell.

Child Products

The ID of the Child products for the offer. This only appears when the offer is a bundle, cross-sell, or up-sell.

Child Categories

The ID of the category (or categories) associated for the offer. This only appears if the offer is set up for one or more categories.

Start Date

The first day when the offer is valid and can be redeemed on your store. All offers must have a start date (or be set to start immediately). If you set a Start Date that occurs in the future, you will not be able to preview the offer until that date is reached.

End Date

The last day when the offer is valid and can be redeemed on your store. You do not have to set an End Date for an offer. If you do not set an end date the offer can be redeemed on your store until you make it stop by retiring it. When the end date for an offer is reached, it is automatically given the status of "expired" and can no longer be redeemed on your store.
