About Creating Styles

When you create a style, the name and description you enter are only used internally in the Command Console to identify the style and are never seen by shoppers on your store.

There are four tabs that are used to define the design in a style. A summary of the four tabs is given below, with more information about each tab on the following pages.

  • Colors & Fonts – Used to edit the font family, font size, font color, and background color for elements of your store.
  • Buttons – Used to define the images used for various buttons on your store.
  • HTML – Used to customize the HTML code that is used for the header, footer, and gutter of your store.
  • Advanced CSS – Used to edit the CSS used by the style. We recommend only those users with a good knowledge of the CSS language edit the CSS in a style.

Colors & Fonts Tab

The "Colors & Fonts" tab shows information about the fonts and colors for the style. On this tab you can define the font family, font size, color, and background color various text and design elements on your store.

Font Families

A font family is a group of similar looking fonts such as Arial, Verdana, and Sans-Serif. When you create or edit a style, you can enter the name of the font family you want to be used with a style. The families you enter are a prioritized list, meaning that the browser will scroll through the list and use the first font it can display (not all browsers show every font).

Font Sizes

Font sizes are entered in either pixel or point measurements. When you enter a font size you must include either "px" for pixel or "pt" for point after the value to indicate how you want the font size calculated.

Colors

You can set the color for many different textual elements of your store, including links and other text seen on a page. You can also set the background color for tables and other content and pages on your store.

Items that can have their color changed will show a sample swatch of the currently set color for the item next to the field where that item’s color is defined. If you want to modify a color you can either enter a hexadecimal value into the field, or click the color picker icon to select a new color in the color picker.

The Color Picker

Colors are selected using a color picker that opens when you click the color picker icon that appears next to those fields where colors can be set.

In the color picker you can use sliders to define a color, or you can enter values in the RGB or hexidecimal fields to set a specific color. If you use the slider, the color sample box will change to show the color you have defined.

Link Types

You can change the colors used for links on your store. There are three different link types you can change the color for in a style:

  • Link – Default color for an unvisited (unclicked) link on your store.
  • Hover – Color for a link when the cursor is hovered over it.
  • Visited – Color for a link that has been clicked.

The Buttons Tab

The "Buttons" tab is used to change the images that are used for buttons on your store. On the tab you will see the image that is currently assigned to each button type. Keep in mind that the "Buttons" tab may show images for buttons that are not set up to appear on your store because of the design or template used by your store’s pages. To change a button image you simply click on the current button image and use image picker that appears to select the image file you want to be used for the button.

Importing a Zip File of Button Images

If you want to import multiple images to be used for buttons, you can create a Zip® file of the images and import that file into the style on the "Buttons" tab. The images you import in the Zip file will be available for you to select when you edit a button. Keep in mind that any folder/directory structure you create in the Zip file will be respected when the Zip file is imported.

The HTML Tab

The "HTML" tab is used to change the default HTML used by your store’s header, gutter, and footer. You can edit the HTML already used or enter completely new code.

The default HTML for your store wraps the core commerce content (special home page offers, product data, shopping cart, and check out flow elements) of your store, which is pulled in by the <%Content%> tag in the Body Contents. Do not change or delete the <%Content%> tag or the HTML will not work.

Using Tokens in HTML

A token is a tag that inserts specific pre-existing content from your store on the page where the token tag is added. Tokens are created for your store when the store/site is created. Some tokens are available on all stores but most tokens are created for individual stores. Contact your Store Operations team if you have more questions about getting or using tokens on your store.

To add a token, you simply click the link to add a token that is seen while viewing the "HTML" tab. When you click the link, a window appears showing you the available tokens for your store. You simply copy the token tag and paste it where you want the token to appear in the HTML.

The Advanced CSS Tab

The "Advanced CSS" tab is used to change the cascading style sheet (CSS) code used by your store. CSS is a language used to create style sheets for Web sites that use HTML and XML coding. Any changes you make to the CSS will override any global site settings that control the same elements.

Using Custom CSS

If you try to enter custom CSS in a style, you will be prompted to confirm this action before you will be able to enter or change the contents of the CSS field. This is because a lot of how your store appears is controlled by the CSS and small changes to the CSS may have a huge and possibly destructive impact on your store. We strongly recommend that only those with an advanced knowledge of CSS should edit this part of a style.

Note: On the "Advanced CSS" tab you will see tags such as <% PAGE_BACKGROUND_COLOR %> or <% HEADING_SIZE %> in the box where you enter the CSS code. These tags use the definition set for these elements on the "Fonts & Colors" tab.
