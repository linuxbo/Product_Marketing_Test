User Role Definitions

There are many different roles that can be assigned to users in the Command Console. Roles control what menus a user can see, as well as what commands they can see and click on each menu. Roles also dictate what a user can do when using a feature as well as how much they can change or what they can affect.

For more about how user roles work, see About User Roles. To learn how to assign user roles, see Creating and Editing Users.

The table below outlines the roles that can be assigned to users in the Command Console. Some notes about this table:

  • Whether a role is set at the site or company level determines whether the tasks you perform are done just for the selected site, or for all sites within a company.
  • The menu paths will help you identify how to find those features and pages where you can perform the actions the role allows you to do.

Fee Administrator

The Fee Administrator and Fee Approver roles are not assigned in the Command Console (they are assigned in another system called "Big Blue").

These roles provide access to the System Fees and Company Fees features (including the bulk import and export tool) and they can be assigned to both internal and client users.

Users do not need the Fee Administrator or Fee Approver roles to view what fees have been assigned to a specific product.

Fee Administrator

  • Create/edit/delete/deploy/retire company fees
  • Create/edit jurisdictions
  • View system fees
  • Use the bulk import and export tool (company fees only)

Fee Approver

  • Create/edit/delete/deploy/retire company fees
  • Create/edit/delete/deploy/retire system fees
  • Create/edit/delete/deploy/retire jurisdictions or classifications within a fee
  • Use the bulk import and export tool (company and system fees)
