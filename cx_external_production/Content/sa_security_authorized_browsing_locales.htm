Configuring Site Settings: Authorized Browsing Locales

The authorized browsing locales setting is used to indicate which supported locales a shopper (in a specific country) can see once they are already on your site.

For example, if your European site has four locales, you can specify whether a shopper who is visiting your site in one of those locales (keep in mind that the locales a shopper can visit may be determined by your automatic geo-location settings) should be allowed to browse your store in the other three locales. Authorized browsing locales essentially limit a shopper's ability to see your store in one of the other locales supported on your store.

Setting up authorized browsing locales can be useful if you want to limit what other locales (and therefore what products and pricing) a shopper can see because of where they are located. This can reduce the possibility that a shopper can make a purchase from a locale that offers products or pricing at a greater discount than the locale you want them to purchase from.

Who Can View or Edit This Setting

Your ability to view or edit a specific site setting varies as follows:

  • The ability to view or edit a setting depends on the template used by the currently selected site, the type of user you are (internal Digital River user, or client user), and the user roles you have.
  • Some users can view a setting but not change it, and some users cannot see a setting at all. Some users can both view and edit a setting.

The table below outlines who can view or edit this setting. Find the template used by your currently selected site (you can see the template name at the top of the Configure Site Settings page) and then find your user type. Configuring the Authorized Browsing Locales Settings

  1. On the Administration menu, select Site, and click Configure Site Settings. The Configure Site Settings page appears.
  2. Click the Security tab.
  3. In the Authorized Browsing Locales area, click the toggle switch to turn the feature to ON.
  4. Make sure the default locale tab is selected. Once you set up the default locale you will need to set up the other locales if you want to configure your supported locales independently of the default locale. By default, supported locales inherit the settings of the default locale so if you do not want to change this you only have to configure the settings for the default locale.
  5. Click Add Group. The Add Authorized Browsing Group window appears.
  6. In the Group Display Name field, enter a name for the group you want to create. The name should reflect the locales you may add to the group, such as "Europe" or "North America." The name you enter must be unique for the site (you can only use a name once per site).
  7. Click Save. Because changes to most site settings are made immediately when you save, a live change warning message appears. You are prompted to enter your user name to verify that you want to make the change on your live store. Enter your user name and click OK. The Add Authorized Browsing Group window closes.
  8. Select the check box for the group you just created and click Add. The Add Authorized Browsing Link window appears.
  9. For the Type option, select whether you want a shopper who is part of the group to be redirected to a site/locale that is part of your Digital River store, or whether you want to redirect shoppers to another URL you have set up.
  10. In the Site Display Name field, enter the name for your site that you want to appear to a shopper in this group.
  11. From the Locale list, select the locale that the shopper can visit if they are part of the group.
  12. Click Save. Because changes made to most site settings are made immediately when you save, a live change warning message appears. You are prompted to enter your user name to confirm the change to your live store. Enter your user name and click OK. The Add Authorized Browsing Link window closes.
