Reordering Rules

The order that rules appear when listed on the Customize Rules page is also the order in which they will be run against products in the marketStream Common Catalog. You can change the order of your rules so that they run in a way that creates the intended result.

When you reorder rules, you enter an Order value for a rule and any unchanged values update to accommodate your change (however, your change will not "bump" another rule out of it's order position). The order you enter for a rule will become the new order value unless another rule already occupies that order. In that case, the rule you change will occupy the next spot in the order list.

For example, if you have three rules and you change the order value of one to be 10, it is actually given a value of 20 because another rule already has that order value. If you wanted the rule that currently has the value of 10 to be changed, you would change the order value of that rule instead.

Rules are given order values in multiples of ten (10) and each rule must have its own unique value. The rule with the order value of ten (10) will be run first, following by the one with twenty (20), thirty (30), and so on.

To reorder rules:

Role Needed: Catalog ManagerA user role that allows the user access to the catalog and product features. Search this file for the "user role definitions" topic for more information.
  1. On the Catalog menu, select marketStream Channel, and click Manage Rules. The Customize Rules page appears.
  2. In the Order field, enter a value that represents the order you want to set for each rule (you do not have to change the order of all rules, just the ones you want). Keep in mind that the value you enter must be a multiple of ten (10). In the example below, when the rules are reordered they would be listed on the page as follows: Publisher (value of 10), Category (value of 20), Cost (value of 30), and Margin (value of 40).
  3. Click Reorder. The page refreshes and new order is applied to the rules. The changes you have made to the order of the rules are not deployed (and therefore the rules are not run against product in the Common Catalog with their current order) until you deploy them. Click Deploy Changes in the taskbar to deploy your rules. See Deploying Rules for more information.
