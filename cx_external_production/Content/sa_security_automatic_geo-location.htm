Configuring Site Settings: Automatic Geo-Location

The automatic geo-location setting allows you to specify the specific site and locale shoppers should see based on their geographic location. The feature also let's you restrict shoppers from certain locations from viewing or shopping on your store. For more information about how automatic geo-location works, see How Automatic Geo-Location Works.

Who Can View or Edit This Setting

Your ability to view or edit a specific site setting varies as follows:

  • The ability to view or edit a setting depends on the template used by the currently selected site, the type of user you are (internal Digital River user, or client user), and the user roles you have.
  • Some users can view a setting but not change it, and some users cannot see a setting at all. Some users can both view and edit a setting.

The table below outlines who can view or edit this setting. Find the template used by your currently selected site (you can see the template name at the top of the Configure Site Settings page) and then find your user type. Use the table to determine whether you can view or edit a configuration setting. Keep in mind that the type of user required to view or edit the setting.

Setting the Automatic Geo-Location Settings

  1. On the Administration menu, select Site, and click Configure Site Settings. The Configure Site Settings page appears.
  2. Click the Security tab.
  3. In the Automatic Geo-location area, for the Unrecognized Country option, select what you want to happen when a shopper in a location that is unrecognized attempts to visit your store.
    • Stay on this site â€“ Shoppers visiting your store from an unrecognized country will stay on the site they are trying to view. For example, if a shopper from an unrecognized country tries to access the en_GB locale of your store and you don't have a rule set up to specifically redirect shoppers in unrecognized countries, they will see the en_GB locale of your store.
    • Go to specific site â€“ Shoppers visiting your store from an unrecognized country will be redirected to the specific site and locale you select from the Site and Locale lists that appear when you select this option.
  4. For the Redirect Settings option, select if you want shoppers redirected to a specific site and locale based on their location and browser language setting.
    • Do Not Redirect â€“ Shoppers are never redirected to another site or locale based on the location they are in (as determined by their IP and browser language setting) when they attempt to visit your store.
    • Always Redirect â€“ Shoppers in a specific country with a specific language setting in their browser are always redirected to the site and locale you define for they are in when they attempt to visit your store.
  5. Click Add. The Add Geo-location window appears.
  6. From the Country list, select the country for which you want to redirect shoppers. This should be the location of the shoppers you want to redirect.
  7. For the Browser Language option, select whether you want browser language to be considered when determining where to redirect shoppers. Select Unmatched Language if you do not want browser language to be considered in the redirect. Select the other option and then select a language from that list to define where shoppers in the country you select in the previous step to be redirected if their browser language matches what you select here.
  8. For the Destination option, from the list, select the site locale you want to be used to display your store when the shopper is redirected.
  9. Click Save. Because changes made to most site settings are made immediately when you save, a live change warning message appears prompting you to enter your user name to verify that you want to make the change on your live store. After you enter your user name and click OK, the Add Geo-location window closes.
