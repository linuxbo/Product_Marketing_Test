About Restrict Deployment

The restrict deployment settings are used to control who can deploy the offer. When an offer is set as restricted, it means that only someone with the Merchandising Deploy Manager role (or an internal user) can deploy the offer.

Note: The Restrict Deployment settings only appear when you create an offer if the Restrict Deployment setting has been set to "Only Specific Offers" in Configure Offer Types.

Setting Restrict Deployment

Deployment is restricted in the Offer Controls area of the second step (the Behavior step) of the offer wizard. Keep in mind that these settings will only appear if the Restrict Deployment setting is set to "Only Specific Offers" in Configure Offer Types.

When you select the Restrict to Merchandising Deployer check box, additional options appear so you can choose how you want to restrict deployment:

  • Always Restrict Deployment – Deployment of this offer will always be restricted to a user with the Merchandising Deploy Manager role.
  • Restrict Until After First Deployment – Deployment of this offer will only be restricted until the offer is deployed for the first time. Once the offer is deployed the first time, anyone will be able to deploy the offer.
  • Restrict Until Specific Date – Deployment of this offer will only be restricted until a specific date. After that date occurs, anyone will be able to deploy the offer.
