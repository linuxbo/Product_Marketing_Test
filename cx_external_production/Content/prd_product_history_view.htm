Viewing Product History

You can view the history of changes to a product and see who made the change and when. This includes changes made to the product using the Command Console, the Import and Export feature, and the bulk XML feed. Viewing history for a product can be helpful when you want to know when a specific change was made.

Note: There are some events that are not logged in product history due to some system limitations: changes to fees, digital rights, and how often subscription notifications are sent.

Understanding What You See

When you view history for a product there are several different pieces of information you see on the screen that can help you understand what changes were made to a product, how, and by whom.

Element Definition
Change Type

Indicates the type of event that was logged:

  • Took Ownership – Indicates that editing ownership of the product was taken by the user indicated in the Modified By column. Ownership simply means that the user was the last person to edit it and no other user will be able to edit the product until they take editing ownership. Ownership is used to prevent two users from making changes to the same product at the same time.
  • Save – Indicates the product was saved. If "(Live Change)" appears after save it means that the change was made immediately to your store (versus a change that requires you to deploy the product before the change is seen on your store).

    When you click the + to view the details of a save event, you can see the following (not all these columns will appear with every save event):
    • Locale – Indicates the locale(s) in which the change was performed.
    • Area – Indicates the general area of the product editor the change was performed. For example, if you make a change to the product name, the area would list "Storefront Settings" as this is where the name can be found.
    • Field – Indicates the field that was changed. Use this name with the Area to help you locate the attribute in the Command Console UI.
    • Old Value – Shows what was entered/existed before the change. If not value was present "No content" will appear.
    • New Value – Shows what was entered/selected with the change.

Note: Keep in mind that just because a product was saved doesn't always mean the product was changed. If someone views the product and clicks the Save button without making any changes, the system will still log that change in the history.

  • State Changed – Indicates that the status of the product was changed. There are many ways that the state a product can change, the most significant is when you edit a product. State changes also happen when you deploy or retire a product.
  • Copy – Indicates that the product is a copy of another product. When a copy event is shown in the history, the product ID and name of the product from which the new product was copied is also shown.
  • Revert – Indicates that the product was reverted to a previous version (and indicates what version that was). For more about reverting a product, see About Reverting Changes to Products.
  • Variation Added – Indicates a variation was added. The variation ID, as well as the varying attributes will also appear in the Change Type column.
  • Variation Deleted – Indicates a variation was deleted. The variation ID, as well as the varying attributes will also appear in the Change Type column.

Product/Variation ID

Indicates the product ID or variation ID of the product in question.

  • If the product is an individual product only the product ID will appear.
  • the If the product is a base product, "(Base)" will appear after the ID.
  • If the product is a variation, the name of the variation as well as the varying attributes (in parentheses) will appear after the ID.
Modified By

Indicates who performed the event, and how they performed the event.

  • If the event was performed by the user in the Command Console, the Modified By column will simply show their user ID and name (in parentheses).
  • If the event was performed by the user using the Product Import and Export feature, the Modified By column will show "(Bulk)" after their user ID.
  • If the event was performed by the bulk XML product feed, the Modified By column will show "Bulk Feed" and then the ID for the XML job in parentheses.
Modified On Indicates the date and time when the event took place.

 

To view product history:

Role Needed: Catalog ManagerA user role that allows the user access to the catalog and product features. Search this file for the "user role definitions" topic for more information.
  1. On the Catalog menu, select Products, and click Manage Products.
  2. Select the product you want to view history for and click View History. The Product History page appears.

Tip: You can also click View History when viewing the details of a product to see the product history.

  1. In Filters area, in the From Date field, enter the start date (MM/DD/YY) for the history you want to view. Then, in the To Date field, enter the last day of changes you want to view. Click Search to show the changes to this product for the time frame you have defined.
  2. When you are done, click Done to return to the Products list or the page from which you opened the Product History page.
