Creating Custom Links

The Links tool is a quick and easy way to create basic links for your store. However, if you need a link that does more that what the tool offers, you create your own link manually. The only requirement is you follow the same structure and format used by links created by the Links feature.

Tip: We recommend that you use the Links tool to create any links you will send to shoppers via email or post on your store or other online site. Using the tool will ensure that the link will behave as expected. If you do build your own link, we strongly recommend you test the link to ensure it works as intended.

How to Build Your Link

To create a link you simply replace the elements shown in bold in the example in Basic Link Structure with the appropriate elements from the tables in the following sections.

For example, to create a "buy" link where a product is added to the Shopping Cart, you would replace "action" with the word "buy" and replace "parameter.value" with the word "productID" and the actual product ID for the product you want the link to add to the Shopping Cart.

Site ID

Every link must contain a site ID. Some companies may have more than one site or store, so it is especially important for those companies to include the correct site ID in their links. However, even companies with only one site or store must include their site ID in the URL or the link will not work.

Locale

Including a locale code in the URL is optional, but we recommend you always include one to ensure the shopper is getting the right version of your store. Locales are added to the URL using the ISO four-character locale code. These are the same codes you should have seen throughout your store and the Command Console. You can only include one locale code in each link.

Action

There are many different actions you can use with a link to do something on your store. The most used action, by far, is the "buy" action. This action triggers a product to be added to the cart. The following table outlines the actions you can use with links.

Action

Description

buy

Adds one or more products to the Shopping Cart. This action replaces all previous "AddItemToRequisition" actions that have been previously used within the Command Console.

This action requires the "productID" parameter. If the product has variations, the variation product ID must be used. You can also include the quantity parameter if you want to have more than one of the product added to the Shopping Cart.

cart

Takes the shopper to the Shopping Cart.

cat

Takes the shopper to the category listing page.

This action requires the "categoryID" parameter.

contact

Takes the shopper to the Contact Us page

global

Takes the shopper to the locale selection page.

help

Takes the shopper to the Help pages.

home

Takes the shopper to the store Home page.

html

Loads an custom page that has been uploaded to your store.

This action requires the "pbpage" parameter.

list

Takes the shopper to a product listing page.

login

Takes the shopper to the login page for your store.

pd

Takes the shopper to the Product Details page.

This action requires the "productID" parameter.

Parameters and Variables

Depending on what action is included in the URL, one or more parameters may be needed to tell your store exactly what you want the link to do. The following table outlines the parameters that can include in your URL.

Parameter

Description

Variables

backlink

Used with the "buy" action to indicate where you want the "Continue Shopping" button in the Shopping Cart to go when clicked.

This parameter should only used with the "buy" action.

Any valid Web address that has been URL encoded.

categoryID

Used to indicate which category you want to show the Category Listing page for.

Any valid category ID for the store.

clearCart

Used with the "buy" action to removes any products that are in the cart at the time when the link is clicked. Does not clear out products that are added via the same link.

This parameter should only used with the "buy" action.

yes

no

currency

Indicates which currency you want to be used to display pricing on the store.

Any valid three-digit ISO currency code can be used with the "currency" variable.

externalRefID

Used with the "buy" action and when you want to use the External Reference ID that you have entered for a product, instead of the system-generated Product ID used to identify which product should be added to the Shopping Cart.

This parameter should only used with the "buy" action.

Any valid External Reference ID set for a product on the store.

forceProduct

Used with the "buy" action to force the product to be added to the cart, bypassing any interstitial pages or forcing the shopper to select a variation (if present).

When used with products with variations, the first variation listed in the drop-down list on the Product Details pages will be used.

yes

nextPage

Used with the "buy" action to indicate what page you want to load when the shopper clicks the link and the product is added to the Shopping Cart.

HomePage

ShoppingCartPage

pbpage

Used with the "html" action to indicate which Custom Page you want the link to display.

Any valid Custom Page name.

productID

Used with the "buy" or "list" actions to indicate which product you want the link to add to the Shopping Cart or show on the store.

Any valid product ID for a product on the store.

programID

Used with the "buy" action to indicate what program should be used to track traffic on the link.

When a link with a program ID is clicked, the visits, page counts, and revenues for each unique program ID will be tracked.

Any valid program ID. The program must be created and deployed on the store.

quantity

Used with the "buy" action to indicate how many of the product should be added to the Shopping Cart.

Any valid positive number.

showCouponCode

Used with the "buy" action to show the Coupon Code entry field in the Shopping Cart when it normally is hidden or not shown for the store.

This parameter should only used with the "buy" action.

yes

no

themeID

Used to indicate what theme you want to be used when the link is clicked.

Any valid theme ID. The theme must be created and deployed on the store.
