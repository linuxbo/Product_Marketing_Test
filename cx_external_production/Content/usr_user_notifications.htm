Viewing and Resending Notifications

When you edit an active user, you can view the notifications sent to the user, and if needed, resend a notification. Notifications can include events, such as the user requesting to be sent their password.

When you view notifications you can see the user's name, the email address to which the notifications are sent, and the notification history. The notification history provides a listing of each notification and provides other summary details.

To view and resend user notifications:

Role Needed: User ManagerA user role that allows the user to create and manage users, including assigning user roles to users. Search this file for the "user role definitions" topic for more information.
  1. On the Administration menu, select Users, and click Manage Users. The Users list page appears.
  2. Click on the User Name of the user you want to view or resend notifications for. The Edit User page appears.
  3. Click the Notifications tab. On this tab you see the following three areas:
    • The User Information area shows a summary for the user you are viewing.
    • The Notification History area shows the notifications sent to the user.
    • The Delayed Notifications area shows any notifications that have not yet been sent to the user.
  4. In the Notification History area, click the notification name of the notification you want to view or resend. The Notification Details window appears.
  5. View the details for the notification. To resend the notification, click Resend. When you are done in this window, click Done to close and return to the Edit User page.
