Style Statuses and Versions

As you work with styles, the status and version will change depending on the actions you take on the style. Understanding how the statuses and versioning works can be helpful when managing your styles over time. Styles can have one of the following four statuses:

  • Design – Indicates the style has changes that have not been deployed on the store. When you first create a style it is saved in the "Design" status and will not be seen by shoppers on your store until it have been deployed.
  • Deployed – Indicates that the style has been deployed to the store and the version does not have outstanding changes waiting to be deployed.
  • Retired – Indicates the style has been retired and will not appear on your store even if a theme that is active on your store is set to use that style.
  • Pending Deployment – Indicates the style has been set to deploy but the style has not been fully deployed on your store. Styles usually only have the "pending deployment" status for a small window of time, usually just a few minutes.

Versions and Modified Dates

When you are viewing an existing style, you will see information about the version number and last modified date for the style near the top of the page. Depending on the style, this area may also show what themes the style has been applied to and when the style was last deployed.

The version number for a style simply indicates how many changes have been made to the style since it was first deployed.

When you edit a deployed style, the changes are saved with a new version of the style. You will then have to deploy the new version (which will be in the "Design" status) for those changes to be seen on your store. This is true regardless of whether the style is part of an active theme or not.

If a style that has changes that have not yet been deployed is used by a theme, the theme will use the last deployed version of that style until you deploy the latest changes. Once a style has been deployed, any changes in that version become part of the most current version of the style.

Loading a Previous Version of a Style

The last five versions of a style are saved. If needed, you can select a previous version of a style while editing the style to load the older version on the page. Once the version is loaded, you can save the version and deploy it, essentially "rolling back" the style deployed on your store to the version you selected.
