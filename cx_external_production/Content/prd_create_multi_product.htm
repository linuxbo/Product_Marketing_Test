Creating Products (with Variations)

The Create Product wizard is used to create new products in your catalog. The wizard walks you through the process step by step, showing only those options and fields that are relevant to the product you are creating.

The product setup for your company determines what fields and default values are seen when creating or editing a product. This means that the information in this topic may not reflect what you see on your site. Contact your Store Operations team to learn more about product setup and default values.

About Variations

Variations are different versions of the same product. For example, a software product can have two different variations, one for the Mac version and one for the PC version.

When you create a product with variations, you first define the "base" attributes for the product (and variations). This includes supported locales, delivery types, store images, and so on. When you create a variation, the base attributes are copied to the variation and if you do not define some attributes before you create a variation, the variation will not have those attributes. You should define as many base attributes as you can before you create your variations.

After you create variations for a product you can customize the attributes and options for each variation. For example, you can set a different price for one variation, or select a different product image for another.

To create a product (with variations):

Role Needed: Catalog ManagerA user role that allows the user access to the catalog and product features. Search this file for the "user role definitions" topic for more information.

  1. To create a new product, on the Catalog menu, select Products, and click Create Product.

  2. In the Template area, select Multiple Product (with Variations) and click Next to move on to the Supported Locales step.

  3. Select the Supported Locales for the product. The default locale of your store (also called the base locale) is already selected by default. To change the default locale for this product, select the Default option next to the locale you want to set as the default. This change will not affect the global default locale settings for your store.

    More about supported locales... Supported locales are used to enter localized content for products. A locale indicates language, country, and currency. For each locale you select you can enter product descriptions and prices in that locale's language and currency. When shoppers view the product on your site in one of the supported locales, the localized product content you entered (descriptions, prices, and so on) can be seen.

    Select the Restrict Product Visibility and Purchasing to Locales Below check box if you want the product to appear (and therefore be purchasable) only in the locales you select. When you get to the step where you define the product, a tab will appear for each of the locales you select here so you can enter localized content.

    When you are done, click Next to move on to the Catalog and Categories step.

  4. Select one or more Catalogs and Categories to add the product to. When you are done, click Next to move to the Product Options step.

    • When you select a catalog, categories within it can be selected.
    • Price lists are associated with a catalog so the catalog(s) you choose determine the price lists you can use with the product later.

  5. Select the Product Options for the product. The options you select here will be configured in the Product Settings step. You will also have the option to vary the product by the options you select here. When you are done, click Next to move on to the Delivery Type step.

    Click to see definitions for the Product Options:
    • Channel (marketStream) – This option only appears if you are participating in the marketStream Intersite program and is used when you want to add the product to the marketStream Common Catalog so that it can be sold by affiliates.
    • Digital Rights – Used when you want to use digital rights (serial numbers or unlock codes) with the product.
    • Game – Used when the product is a game and you want to show things like game ratings on your store.
    • Gifting – Used when the product can be purchased as a gift.
    • Preorder – Used when the product can be preordered before it is released on your site. You can set the release date and other options.
    • Software – Used when the product is software and you want to specify the software platform, system requirements, or enter a product-specific end-user license agreement.
    • Subscription – Used when the product is a subscription product. Subscription products are products that shoppers get in regular intervals on order. You can set the length of the subscription, the renewal options and settings, and whether the subscription has a free trial.
    • Trial – Used when the product will be distributed with a trial period for shoppers to "try before you buy."
    • Volume Licensing Pricing – Used when you want to use a volume license plan with the product.

  6. Select the Delivery Type(s) for the base product and variations. If you select more than one type you must vary the product by a delivery type in a later step (a variation can then be created for each type you select). When you are done, click Next to move on to the Product Settings step.

    Delivery type is how the product is fulfilled, that is how shoppers will receive the product after they purchase it. You will configure the specific options for the delivery type(s) in a later step.

    Click here for definitions of the Fulfillment Methods:
    • Download – Used when you want shoppers to download the product when they purchase. You must have the file(s) you want shoppers to download on your computer so you can upload them to the Command Console when you create a product of this type.
    • Physical – Used when you have a physical version of the product that will be sent to shoppers when they purchase. Physical product orders can be fulfilled by Digital River or a third-party fulfiller.
    • Physical on Demand – Used when you want to have a physical version of the product that is created on-demand (when the order is placed) sent to shoppers. This option will not appear if you do not have a contract with a Physical on Demand fulfiller already set up.
    • Other – Used when you are not using any of the methods listed.

  7. The Product Settings step is where you enter details for the product. Required fields are marked with a orange asterisk and must be completed before saving or moving to the next step. Most of the fields and options on this page vary based on the options and delivery type(s) you selected in previous steps. For example, if you selected "Digital Rights" as an option for the product, the Digital Rights area appears. Your site's template may also affect what areas you see.

    The first group of fields and options in the Product Settings area are used to enter basic information for the product:

    • In the Internal Product Name field, enter the name of the product that is used internally by the system. The Internal Product Name is limited to 255 characters.
    • In the Site Product Name field, enter the name of the product as it will appear to shoppers on your site. The Site Product Name is limited to 255 characters.

    The text you enter here is searchable by shoppers on your site. Read more... When a shopper searches your site, the text you have entered for the Site Product Name, Keywords, and Short and Long Description are what is searched and used to generate the results. For example, if you enter "software" as one of the keywords for a product, a shopper who searches for "software" will get the product in their results.

    • In the SKU field, enter the code (letters, numbers, symbols, or combination) that uniquely identifies the product. Unlike the Product ID, which is generated by the system, your company decides the SKU and how it is used to identify the product.
    • For the Options, clear (un-check) the Display Product check box if you do not want the product to appear on your store. Clear (un-check) the Purchasable check box if you do not want the product to be able to be purchased on your store. By default these options are selected because they are usually selected for products.

    For example, you would clear (un-check) the Display Product check box if the product was part of a marketing offer that you wanted to provide to a select group of shoppers, or clear (un-check) the Purchasable check box if the product is not available for purchase until a certain date, such as a major revision release date.

    • In the External Reference Number field, enter the code (letters, numbers, symbols, or combination) that uniquely identifies a product to a third party. This value is not localized for a variation. More... Unlike the Product ID which is generated by the system, or the SKU which may not be unique, a third party can make the decision about what makes this product unique. The External Reference Number is set only once per product. It can be edited, but it is not localized or modified for a variation. When you import products, it can be set in the product's default locale only.
    • In the Manufacturer Name field, enter the name of the company that manufactured the product. This value can be customized and localized for a variation.
    • In the Manufacturer Part Number field, enter the number the manufacturer uses to identify this product. This value can be customized and localized for a variation.
    • In the Tax area, from the Tax Group list, select the group that most represents the type of product. Then, from the Tax Type list, select the type that most represents what your product actually is. If the available tax types do not accurately represent your product, contact your Store Operations team. 

    Tip: You can view a list of all available Tax Groups and Tax Types (including the SABRIX™ code used by each) by clicking View Tax Settings. This may be helpful if you are unsure which Tax Group to select, or what Tax Types are included in each Tax Group.

    Tax calculations are based on a variety of factors... including ship-from address (product source), ship-to address (product destination), product weight, and the product's categorization. We use a highly respected and powerful system to calculate tax on orders. By creating the tax groups and tax types, we have made it easier for you to classify your product so that our tax system can accurately calculate tax no matter where in the world the order is placed.

    • For the Return Type select the method by which the product will be returned. Select Physical Product if the product is a physical item that will be returned to you, or Electronic Letter of Destruction (eLOD) for digital products (products fulfilled by download) that will use a eLOD. If you do not want to require either of these two options for a product to be returned, select Nothing Required.

    The Product Details Page tab is where you enter information about the product that will appear on the Product Details page on your store. See example of tab.

    • In the Short Description field, enter the short description for the product. The short description is a summary of the product that appears on your storefront if your site or pages are set to display it. By default it is set to appear on the category product listing page, the product search results page, and the product details overlay that appears in the shopping cart.

      You can use HTML in the description fields to format the appearance of the information entered. See Using HTML in the Command Console for more about using HTML in these fields.

    The text you enter here is searchable by shoppers on your site. Read more... When a shopper searches your site, the text you have entered for the Site Product Name, Keywords, and Short and Long Description are what is searched and used to generate the results. For example, if you enter "software" as one of the keywords for a product, a shopper who searches for "software" will get the product in their results.

    • In the Long Description field, enter the long description for the product. The long description appears to the shopper on the product detail page, if the page is set to display it.

      You can use HTML in the description fields to format the appearance of the information entered. See Using HTML in the Command Console for more about using HTML in these fields.
    • In the Keywords field, enter the keywords (separated by commas) for the product.

    The text you enter here is searchable by shoppers on your site. Read more... When a shopper searches your site, the text you have entered for the Site Product Name, Keywords, and Short and Long Description are what is searched and used to generate the results. For example, if you enter "software" as one of the keywords for a product, a shopper who searches for "software" will get the product in their results.

    • Click Select next to the Thumbnail Image and/or Product Image fields and select an image to use for the product in the Select an Image window that appears.
    To use the Select an Image window:

    The Select an Image window is used to select an image, import a new image, and create folders in which to organize your images.

    In this window you will see the images available for use on your site. The file name and size are shown for each and when you select a file in the list a preview of the image appears on the right. The width and height (in pixels) for the image are shown under the preview for easy reference.

    Note: The exact size of the image you use for the category thumbnail, product thumbnail and product image may vary by the unique set up of your store, however most thumbnail images are 100px × 100px product images be 250px × 250px.

    • You can create folders to organize your files. Click New Folder and enter a folder name in the window that appears. You cannot use spaces and special characters in folder names, but hyphens (-) and underscores (_) are allowed.
    • To select a file to use, click the option button next to the image and click Add Selected.
    • To import a new file, click New File. The New File window appears. Click Browse to select the file you want to import. Select the Overwrite Existing File with Same Name if Present check box if you want the file to be saved over an existing image with the same name (useful when updating existing buttons with changes). Click Upload to import the file. When the import is complete you are returned to the Select an Image window.

    The Email Settings tab is where you enter the extra text that you want to appear in the confirmation emails sent to shoppers when they purchase from your store. See example of tab.

    • In the Text-Based Email field, enter the raw text you want to be added to the email sent to shoppers after an order is placed. This text appears unformatted.
    • In the HTML-Based Email field, enter the text (with HTML formatting/tags) that you want to be added to the email sent to shoppers after an order is placed. See Using HTML in the Command Console for more about using HTML.

    The Checkout Settings tab is where you indicate the checkout fields you want to appear this product is purchases and any extra text that you want to appear on the Thank You page a shopper sees when they place an order. See example of tab.

    • In the Checkout Fields area, select the fields you want to be required when this product is purchased. Depending on the fulfillment method for the product, some of the check boxes may be selected by default and disabled (greyed out) because they are always required for the type of product.
      • The Billing Address check box is always selected and disabled because a billing address is always required.
      • The Shipping Address check box is always selected and disabled if you selected Physical or Physical on Demand as the Fulfillment Method.
      • The Shipping Method check box is always selected and disabled if you selected Physical or Physical on Demand as the Fulfillment Method.
    • In the Thank You Page area, in the Thank You Message field, enter the text that you want to appear on the page shoppers see after they place an order.

    The Minimum/Maximum tab is where you define the minimum number and maximum number of this product a shopper can purchase in a single order. If there are no quantity limits for shoppers, enter zero (0) in the fields or leave them blank. See example of tab.

    • In the Minimum Quantity field, enter the minimum number of this product a shopper must purchase.
    • In the Maximum Quantity field, enter the maximum number of this product a shopper can purchase (in a single order).

    If you selected additional locales for the product to support, a tab will appear for each of the locales you selected. Use the information about to define the product for those supported locales. You will not be able to move to another locale tab is the required fields have not been completed on the Default locale tab.

    The product information you enter on the Default locale tab "copies down" to the supported locale tabs until you select that locale to "override" the default settings. This means that if you have entered all the information you need for the product on the Default tab before you customize the content for supported locales, you only have to change the information you want to be localized for the specific locale. Keep in mind that once you "override" the default locale settings, any changes you make for the Default locale will not copy down to the locale.

    • To enter localized content for the product, click one of the locale tabs at top.

    • When you click a locale tab, you muse select the Override Default Locale Settings check box to enter locale-specific content on the tab.

  8. The remaining areas on the page are used to enter the settings for the base product. The areas that appear do so because of the options you chose in the Product Options and Delivery Type steps. Use the information in Product Options to help you enter the settings for the base product.

    9. Note: The settings you enter or define are for the base product only, but many base product attributes "copy down" to variations when they are created. This means you should only enter information you want to be used for the base product and variations you will create. More...When you create a variation, the variation inherits the Product Settings for the base product. This inheritance is not recursive, that is, if you make a change to the base product details after a variation is created the variation will not be changed. After a variation is created you can edit the variation details and customize the settings and details for the variation as needed. You can also enter base product details that you did not want to copy down to the variations you created.

  9. When you are done entering the product details and settings, click Next to move on to the next step.

    • When you click Next, the product information you have entered in validated and temporarily saved. The product is not fully saved until you complete the wizard.
    • If you did not complete all required fields, or if an error is found in the information you have entered you will not be able to move to the next step until the problems are fixed.

  10. Select how you want the product to vary in the Variation Options. This is how the variations will differ from the base product attributes you have already entered for the product. When you are done, click Next to continue to the Add Variation step where you will create the specific variations of the product.

    • The choices you make here are called the "varying attributes" or "vary by" attributes for the product.
    • The varying attributes you see are determined by the selections you made in the Product Options and Delivery Type steps.
    • Click Previous to go back to change your selections and what you can vary by, if needed.
    • Some Product Options can be varied by different attributes of that option. For example, if you chose Subscription as a Product Option, you can vary by Duration or Renewal Date. The Product Option is listed in parentheses for these varying attributes. Click to see an example of what appears if a product is being varied by the Software attribute of Platform and the Subscription attributes of Duration and Renewal Date Basis.

  11. Click New to create a variation. The Create Variation window appears.

  12. The attributes you can vary on appear in lists in the Create Variation area. Select the attributes for the variation you want to create from each of the lists that appear and click Create or Create & Edit. Repeat as needed until all the variations you want are created.

    • Click Create if you want to create the variation but not define or change the variation details at this time. This is helpful when you have several variations you want to create at the same time, or if want the variation to use the same base attributes you defined for the product. When you click this button, the variation is created and appears in the Variations area at the bottom half of the window. Click to see an example of what appears if two variations were created with the same Platform attribute (1 User License) but a different Delivery Type (Download vs. Physical on Demand).
    • Click Create & Edit if you want to create the variation and edit the variation details right now. When you click this button, the variation is created and the Edit Variation page appears. On this page you can change what you entered for the product base attributes for the specific variation being created.

    When you are finished creating the variations, click Done or Save. You are returned to the Edit Variations page.

  13. When you return to the Edit Variations page, if you want to change the order of the variations, click Ordering. The Order Variation Attributes page appears.

    The variation order determines the order in which variations appear on your site. For example, this may be the order of the variations that appear in a drop-down list.

    • Click an arrow icon in the Sort Order column to move a variation up or down one position in the list.
    • If you click an arrow for a folder item it will move all attributes in that folder up or down one position in the list.
    • When you are done, click Save to save the order you have set and return to the Edit Variations step.

  14. Click Set Pricing to enter pricing for the product. The Configure Pricing window appears.

  15. If you are done setting up the product, click Finish to validate the product and save.

    • If the validation and save occurs successfully, the Products list page appears with a message indicating the success at the top of the page. You can click the success message to open the product detail page and edit (or view) the product you just saved.
    • If errors are found during validation, you will be prompted to fix the errors before the product will be saved.
    • If warnings are found during validation, the product will be saved but you are notified of the validation warnings in messages at the top of the Products list page.
    • If you want to go back and change information you have entered, click Previous until you get to the step with the information you want to change.
    • If you want to cancel creating the product, click Cancel and all the information you have entered is thrown away.
