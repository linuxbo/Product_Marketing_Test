Unlock Code Attributes

The following table outlines the attributes that can be set when unlock codes are enabled for a product.

Attribute

Definition

Generation Method

Indicates how the unlock codes will be distributed or generated for the product. This is not how the codes themselves are generated (they are always generated using a text file that you import).

The options you see in the Generation Method list will differ based on how digital rights have been set up to work on your store. Only the two most common options are explained here. Contact your Store Operations team for more about the options you see for your store.

Fixed Number – Used if you want all purchases of the product to use a single unlock code. When you select this option, the "Fixed Number" input field appears in which you would enter the unlock code all shoppers will use for this product.

Manual List – Used when you will enter or import a list of unlock codes to be used by the product. If you select this option you will have to set the other options that are explained in this table.
External – Use when you have created a custom integration to deliver unlock codes.

Upload ARM File (Used only with SoftwarePassport generation methods)

Upload New ARM File

Used to find the ARM file with the keys for the product.

Manual ARM Settings (Used only with SoftwarePassport generation methods)

Key Type

Version of keys that can be used with the product.

Encryption Template

The encryption key for the product.

Encryption Level

Signature level of the key.

Hardware Locking Level

Hardware locking level chosen in SoftwarePassport.

Maximum Reissues

Total number of times an unlock code can be reissued.

Maximum Activations

Total number of times the product can be activated with an unlock code.

Project ID

(Optional) The Armadillo project ID.

License To

Indicates how the key should be licensed.

Use Digital River Edition Keys

Indicates that you want to use Digital River keys with the unlock codes.

Enable Multiple Physical Activations per Serial Number

Indicates that multiple activations are allowed for the same key.

Store Expiration Value in Key

Indicates that the expiration date should be included in the key.

Country

The country for the keys used. This attribute is only set when the country-based distribution method is used.

Maximum Days of Extension

The number of days allowed for an extension. This attribute is only set when the country-based distribution method is used.
