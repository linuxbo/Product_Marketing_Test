About Locking and Unlocking

The locking/unlocking feature helps protect certain Store Builder items from being changed by two users at once. When an item is edited it is locked and must be unlocked before any other user can edit the item again.

Enabling the Locking Feature for a Site

The locking feature must be enabled for a site before items can be locked. The locking feature is enabled in the site settings, on the Features tab. See About Configuring Site Settings for more information.

Note: If you want to disable the locking feature after it has been enabled, you can only do so if all items are unlocked in Store Builder.

Items that Can Be Locked

There are several items in Store Builder that use the locking feature:

  • Themes
  • Styles
  • Custom pages
  • Content layouts

How an Item Becomes Locked

An item automatically becomes locked when it is edited. To edit an item you simply click the Edit button when you are viewing the details for an item. An example of an Edit button appears below.

  • Locking happens automatically; you do not do anything specific to lock an item except edit it (click the Edit button).
  • Once you are taken to the page where you actually edit the item, if you click Cancel or navigate to another page without clicking Apply or Save the item remains locked.

Note: Is is important to note that clicking the Edit button is what locks an item. This is especially important to note for deployed versions because if you click the Edit button but then make no changes (click Cancel or navigate to another page without clicking Apply or Cancel) the deployed version of the item is locked.

What Happens When an Item is Locked

  • When an item is locked, the person who locked the item (or someone with the Storefront Supervisor role) is the only person who can edit, deploy, revert, retire, or unlock the item.
  • When an item is locked, users who didn't lock the item can only copy or preview the item.
  • Locked items show a lock icon when they are shown on the list page for that item.
  • Locked items can be viewed by other users but they cannot be edited, deployed, reverted, or retired.
  • Locked items can be copied.
  • Locked items can be previewed.
  • Deploying or retiring a locked item clears the lock. (Only the person who locked the item or a person with Storefront Supervisor role can deploy or retire a locked item.)
  • History for locked custom pages and locked content layouts can viewed.

Unlocking Items

Locked items can only be unlocked by the person who originally locked the item, or by someone with the Storefront Supervisor role. If the person who locked an item deploys or retires that item, that also unlocks the item.
