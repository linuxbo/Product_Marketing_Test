Creating Marketing Offers

Marketing offers are promotions or discounts used to entice shoppers to purchase products from your site. There are many types of offers you can create including bundled offers that tie two or more products together. See About Marketing Offers for more about the types of offers you can create.

The steps to create different types of offers are similar. Editing an offer is similar to creating one, except you do not use the Offer Wizard and instead edit the parts of the offer separately.

The specific steps you take to create an offer will vary, based on the type of offer you are creating and the additional options you want to enable or use with the offer. The steps below provide a summary of the steps and link to other topics in this file that explain what to do or select in those steps in more detail.

Opening the Offer Wizard

Role Needed: Merchandising ManagerA user role that provides access to various merchandising features, including offers and tracking programs. Search this file for the "user role definitions" topic for more information. and Merchandising Discount ManagerA user role that allows the user to set the discount amount or discount percentage in marketing offers. Merchandising Discount Managers cannot create new offers, just set the discount for an existing offer. To create an offer you would also need the Merchandising Manager role.
  1. Before you can create an offer, you should know how you want the offer to behave. See Things to Consider Before Creating an Offer for more information.
  2. On the Marketing menu, select Offers and Programs, and click Create Offer.

First Step of the Offer Wizard (Type)

The Type step is where you enter basic information about the offer and choose the type and trigger.

  1. Enter the basics for the offer (Name, Description, External Reference ID). Keep in mind that neither names or descriptions are seen by shoppers on your store.

Note: In order for the External Reference Offer ID to work, the Command Console must be set up to communicate with whatever third party system you use to track offer usage. When an order is placed from an offer that has an external reference ID, the Command Console will send the external reference offer ID to whatever system has been integrated to receive it.

Please note that reports in the Command Console will not contain metrics about usage of offers by the external reference ID. Also, no validation is done to ensure that every external reference offer ID is unique and not used by any other offer for your site.

  1. In the Expiration area, select whether you want the offer to begin and/or end on a particular date.
    1. Select the Start Date check box if you want the offer to begin on a certain date (the offer must be deployed prior to this date for the offer to become live on that date) and then define the start date for the offer.
    2. Select the End Date check box if you want the offer to end on a certain date and then define the end date for the offer. If you set an end date, the offer will automatically become "expired" when it reaches this date.
  2. In the Type area, choose the offer type. The offer type determines how the offer behaves on your store and the types of discounts and products it can provide. You will notice that as you select a type, the selections in the Trigger area below it become available or unavailable depending on the type you choose.
    • See Choosing the Offer Type for more about the type of offer you should create depending on what you want the offer to do.
    • If the type you want to create isn't listed it means it has not been configured to appear as an option for your site (see Configuring Offer Types for more information).
    • This topic explains how to create offers in general, but you can learn more about creating specific types of offers with Creating Coupon Code Offers, Creating Custom Bundle Offers, and Creating Shipping Offers.

Note: Once you complete the offer wizard and save an offer you cannot change the type.

  1. In the Trigger area, choose the offer trigger. Depending on the offer type you selected you may not be able to select some triggers. See Choosing the Offer Trigger for more about the types of triggers and see Choosing the Offer Type to learn which triggers are available with each offer type.

Second Step of the Offer Wizard (Behavior)

The Behavior step is where you define how you want the offer to behave on your store.

Note: What you see in this step will differ based on the type of offer you are creating and the trigger you selected.

Use the information and links below to help you define the settings you see on the screen. Settings are listed below in alphabetical order but the order in which they appear on your screen will differ.

Bundle Options

If you selected an offer type that is a bundle (with the exception of the custom bundle type) this area appears so you can define the relationship between the products in the bundle. Select the options that represent how you want the parent and child products to behave in the offer. See About Bundle Policy for more about the options you can select and how it will affect the offer behavior.

Coupon Codes

If you selected the coupon code offer type, an area will appear for you to indicate how you want to get coupon codes for your offer. Select how you want to add coupon codes to the offer. Once you complete the wizard you will be able to add, generate, or import the codes for the offer. See Creating Coupon Code Offers to learn more about the options you have for adding coupon codes to your offer, and how to add them after you have completed the wizard.today

Discount Level

This area will either have options you can select or show the discount level for the offer type you have selected. Two of the following three possible options will appear:

  • Product(s) (Line Item) – Discount is applied to each individual product associated with the offer, however all products must use the same type of discount (percent off, amount off, etc.).
  • Order – Discount is applied to the total of the order if the order meets the requirements for the offer.
  • Bundle – Discount is applied to the total for the bundle. Other products in the order that are not part of the bundle will not be discounted.

Offer Controls

These settings are used to control how many times an offer can be used, what locales can see and redeem the offer, and who can redeem the offer (public shoppers vs purchase plan shoppers). See About Usage Controls to learn how to control how many time the offer can be redeemed. See About Locale Restrictions to learn how to control what locales can see or redeem the offer.

Offer Display

This area is where the POPs that have been associated with the offer type you selected will appear. A POP determines where and how the offer appears on your store. If you want to use a POP with the offer, select the POP from the list. If the POP you want to use isn't listed, it means it hasn't been added to the offer type in the offer settings.

Tip: Keep in mind that some offers require a specific POP to be selected or the offer will not appear or work as expected. See About Offer Types to learn if the offer type you selected requires a specific POP.

When you select a POP, the Supported Locales area will appear at the bottom of the page so you can choose the locales can see the POP (and therefore the offer). If you want to control what locales can see the offer but are not using a POP, use the setting in the Offer Controls.

Offer Tracking

This setting is used to indicate how you will be distributing the offer to shoppers and is used to track the offer performance in the Merchandising Dashboard. Select how you will distribute the offer and track it on the dashboard.

Shipping Discount

If you are creating an offer of the "discount" type you can optionally set a shipping discount that will apply to the offer in addition to any other discount you define. The types of shipping discounts you can provide are the same as the types of shipping discount offers you can create. See About Shipping Offers for more information.

Subscription Controls

These settings are used to indicate how the offer should behave if subscription products are part of the offer. This area appears for every offer. If you do not associate subscription products with the offer, these settings will be ignored.

  • Apply to Initial Purchase – Offers with subscription products can be used if it is the first (initial) purchase of the subscription by the shopper.
  • Apply to Manual Review – Offers can be applied to orders where the shopper is manually renewing their subscription, if the offer has been created and deployed for the subscription product that needs to be renewed.
  • Apply to Auto Renew – Offers can be applied to automatic renewal orders, if the offer has been created and deployed for the subscription product that is being renewed.

Third Step of the Offer Wizard (Products and Pricing)

The Products and Pricing step is where you associate products to an offer and define the discount (if any). Use the information below to help you set up the products and pricing for your offer.

What you see on the Products and Pricing step will vary depending on the offer type you selected and the discount level you selected in the previous step. If you are creating a custom bundle offer, the Products and Pricing step will be very different. See Creating Custom Bundle Offers for more information.

Pricing Information

The Pricing Information setting is where you choose the currency you want to be used for the offer. This is the currency that will be used to calculate the discount (if the offer provides one) and convert that discount into other currencies. We recommend you select the currency that is the default for the price list used by the products you will add to the offer.

Product Eligibility

The Product Eligibility area is where you define which products a shopper must purchase to be eligible to redeem the offer. The option you select here will determine what appears next on the page.

Non-Bundle Options

The information below lists the options that may appear for non-bundle offers depending on the eligibility option you selected.

  • Specific Products – Select this option if you want to choose products a shopper must purchase in order to redeem this offer. If you select this option, the Products area appears where you can select the products you want to add to the offer and indicate whether you want to provide a discount for the products and whether the discount should be the same for all products or vary from product to product. See example.

  • Specific Categories – Select this option is you want to use a category of products as the eligible products. If the shopper purchases any of the products in any of the categories you add, they will be eligible for the offer. This option makes it easy to add many products to your offer at once. If you select this option, the Categories area appears where you can add the categories of products you want to be used for eligibility and indicate whether you want to provide a discount for the products in the categories. See example.

  • All Products Except Excluded Products – Select this option if you want every product in your catalog to be an eligible product, except for the ones you add to the Excluded Products list (you do not have to exclude products if you want every product in your catalog to be eligible for the offer). When you select this option, the Excluded Products area appears where you add those products that you do not want to be used to determine eligibility for the offer and indicate how you want to discount the products in those categories you do want to use for eligibility and indicate how you want to discount the products you do want to use for eligibility. See example.

  • All Categories Except Excluded Categories – Select this option if you want every product in your catalog to be an eligible product except the products in the categories you add to the Excluded Categories list (you do not have to exclude categories if you want every product in every category your catalog to be eligible for the offer). If you select this option, the Excluded Categories area appears where you add the categories of products that you do not want to be used for eligibility and indicate how you want to discount the products in those categories you do want to use for eligibility. See example.

Bundle Options

When you are creating a bundle offer, the areas that appear where you add products are different to allow for parent products and child products (see About Bundle Policy to learn more about parent and child products in a bundle offer.

Note: The information in this area does not apply to custom bundle offers. See Creating Custom Bundle Offers for more about creating custom bundle offers.

The information below lists the options that may appear for non-custom bundle offers depending on the eligibility option you selected.

  • Specific Products – Select this option if you want to choose the parent and child products a shopper can purchase in with the offer. If you select this option, the Parent Products area appears where you can add the products you want to trigger the offer, and the Child Products area appears where you add the products that are bundled with the parent product. You must also indicate (for both parent and child products) whether you want to provide a discount for the products and whether the discount should be the same for all products or vary from product to product. See example.

  • Specific Categories – Select this option is you want to use a category of products as the parent product. If the shopper purchases any of the products in any of the parent categories you add, they will be eligible for the offer and the child products will be bundled with the offer. This option makes it easy to add many products to your offer at once. If you select this option, the Parent Categories area appears where you can add the categories of products you want to be used as the parent product, and the Child Products area appears where you add the child products. You will also have to indicate (for both parent and child products) whether you want to provide a discount for the products in the categories. See example.

  • All Products Except Excluded Products – Select this option if you want every product in your catalog to be a parent product, except for the ones you add to the Excluded Parent Products list. When you select this option, the Excluded Parent Products area appears where you add those products that you do not want to be parent products, and the Child Products area appears where you add the child product. You will also have to indicate (for both parent and child products) how you want to discount the products that will actually be used in the offer. See example.

  • All Categories Except Excluded Categories – Select this option if you want every product in your catalog to be an parent product except the products in the categories you add to the Excluded Parent Categories list. If you select this option, the Excluded Parent Categories area appears where you add the categories of products that you do not want to be used as parent products. You will also have to indicate how you want to discount the products in those categories that will be actually used for the offer. See example.

Fourth Step of the Offer Wizard (Display Content)

The Display Content step is where you enter the sales pitch, images, and other text that is used by the POP (for the offer itself and for products in the offer) when the offer is displayed on your store. If you didn't select a POP than only the "offer description in cart" display content will appear in this step (the offer description in cart is used by all offer types regardless of POP).

To learn more about display content, see About Offer Display Content. To learn more about the offer description in cart including how it appears on a store, see About Offer Description in Cart.

If you are creating an custom bundle offer, some fields will appear for you to enter names for the groups of products you created for the offer. See Creating Custom Bundle Offers for more information about what to enter in these fields and where what you enter appears on your store.

Fifth Step of the Offer Wizard (Review)

On the Review step of the wizard, a summary of the offer you have defined is shown. You can see what type of offer and trigger you chose, any products associated with the offer, and locale or offer content information. Review the information on this screen and do the following:

  • To make any changes to the offer, click Previous to get to the page with the information you want to edit. Use the information in the steps above to help you make changes.
  • If creating a coupon code offer, an option appears at the bottom of the Review page for you to indicate if you want to add coupon codes to the offer now. Make your selection for this option.
  • When you are satisfied with the offer setup, click Finish to complete the offer. A message appears asking you if you want to deploy the offer now, when it is created. Click OK to create and deploy the offer, click Cancel to just create the offer (you must deploy it at another time for it to appear on your site and be seen by shoppers).
