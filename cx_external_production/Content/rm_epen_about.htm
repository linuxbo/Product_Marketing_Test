About the ePen Report

Every day, Digital River records information about sales and performance on your store. This information is used to generate the ePen (or electronic penetration) report. This document explains what the ePen report is, how to view a report, and what the data and sections of the report mean.

The ePen report is most used by accounting and finance personnel to reconcile Digital River payments and transactions for the month. It is not typically used for marketing or promotion tracking. If you are interested in data about sales, product promotions, product performance, store traffic, and operational data you can generate one of the 60+ reports available in the Online Reporting tool in the Command Console.

Note: To view ePen reports you must have the View ePen Report user role assigned to you in your user profile, and you must have WinZip® installed on your computer. For more information about roles and user profiles, see About User Roles.

What is the ePen Report?

The ePen report contains information about commerce transactions, non-transaction fees, transaction fees, and EDS\CDToGo\Backup CD sales (if your store has EDS or Backup CD) for your store. The report is automatically generated and available for download via the Command Console.

Note: There are different versions of the ePen report that generated for a store. Each version reports on different data (commerce, non-transaction fees, and EDS/CD2Go) and is a summary or detail report. There is also a report the US region and one for all other regions.

There are two times during the month when the ePen report is generated. The first version is a preliminary report that is generated during the first five days of each month for the previous month (for example a report generated during the first week of August will show data for July). The second version is a final version that is generated once a check has been cut and sent to you. You will see both versions in the Zip file you download within the Command Console. If a store has no sales for the month, the reports will not be generated. If needed, you can view or download old reports from within the Command Console.

The ePen report is actually multiple spreadsheet files that can be viewed or edited with Microsoft® Excel or another compatible spreadsheet program. You can have multiple reports, one for each region where your store has sales.

The reports also contain locale information, so you can sort or filter the data in the spreadsheet, based on locale within the region. The following summarizes the different types of spreadsheets that make up the ePen report:

  • Summary and Detail Commerce Report – Outlines transactions in the region for the month. The Summary report provide a single total for all transactions, fees, and so on that occurred for the month. The Detail report provides similar information but for each transaction that occurred on the store for the month. The numbers in this report may not match the numbers you see in the sales feed.
  • EDS/CDToGo Report – A summary report that shows sales, discounts, and fee associated with EDS and/or CDToGo transactions.
  • Non-Transaction Fee Report – Outlines the non-transaction fees paid or due for the month.
