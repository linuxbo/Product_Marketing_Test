Managing Integration Exceptions

The Integration Exceptions tool is used to view and resubmit failed integration requests. There are several different types of exceptions that can occur, with the most common being digital rights and fulfillment exceptions.

When an integration exception occurs, the problem causing the exception must be fixed and the transaction re-extracted. As long as the exception remains unfixed, the client and Digital River cannot charge the shopper and therefore recognize revenue from the transaction. Fixing the exception may require that a line item or the entire order is canceled. Orders should only be canceled when you are sure the problem causing the exception cannot be fixed (canceling a line item or order also causes the client and Digital River to lose revenue). Once a line item or order is canceled it cannot be recovered or "un-canceled."

To manage integration exceptions:

Role Needed: Integration Exceptions Manager or Integration ManagerA user role that provides access to integration exceptions. Search this file for the "user role definitions" topic for more information.
  1. On the Administration menu, select Company, and click Integrations Exceptions. The Manage Integration Exceptions page appears. By default, all exception types appear in a list on the page when it loads. You may need to redefine the search parameters to find the specific order with the exception you want to view. Click here to see an example of the Manage Integration Exceptions page.

  1. By default, all exception types appear in a list on the page when it loads. You may need to redefine the search parameters to find the specific order with the exception you want to view
    1. In the Search area, from the Exception Type list, choose the type of exception you want to find.
    2. (Optional) In the Order ID, Principal ID, or Product ID fields, enter the IDs you know about the order with the integration exception you want to find.
    3. (Optional) In the Fulfiller Part Number or SKU fields, enter the part number or SKU of the product involved in an integration exception.
    4. Click Search. The search results appear on the page.
  2. Click the Order ID for the transaction with the exception you need to fix. The Transaction Detail page appears. This page shows a summary of the transaction and gives information about the exception, including any error codes. This page also contains links to view the order summary or order history.
    • To show the history for the transaction, click Show History in the taskbar. The History table appears at the bottom of the page and shows the history for the transaction.
    • To view the order summary, click the Order ID link near the top. The Order Summary page appears. From this page you can view details about the order, including the customer information and transaction invoice.
  3. Depending on the type of exception, you can fix the problem causing the exception, cancel the line item with the exception, or cancel the order with the exception. Most often you will update information for the product, deploy those changes, and re-extract (resubmit) the transaction to fix the exception. There are a number of scenarios for why a specific exception occurs, so it is not possible to explain all the actions that should be taken to fix each type of exception here. The list below can help you determine the right action to take for a specific type of exception. See About Integration Exceptions for more information about when to determine what action to take.
    • For fulfillment exceptions, look in the Exception column to determine which of the line items in the transaction is causing the exception (this column may also reference what must be fixed). Then, click the Product ID for the product. On the page that appears, you can take action to correct the exception (fix the SKU, External Reference Number and/or Fulfiller Part Number). When you are done editing the product and deploying the changes, you should re-extract the transaction.

    • For digital rights exceptions, if Digital River hosts the DRM for the product, you can update the serial numbers or unlock codes, which will usually fix the problem causing the exception. For example, a digital rights exception occurs when there are no more serial numbers or unlock codes associated with the product. Adding new serial numbers or unlock codes will allow the exception to be reprocessed (make sure any changes to the product are deployed before you re-extract).
    • If you cannot fix the problem causing the exception, or if the exception is because of a bad part number, you can cancel the line item causing the exception. If there is only one line item in the order (and you cancel it) the entire order becomes canceled. If there is more than one line item in the order and you cancel some (but not all) you must re-extract (resubmit) the transaction after canceling the line item(s) in question. To cancel one or more line items, in the Corrective Action Taken or Removal Reason field, enter the reason for the cancellation. Then, select the line item(s) and click Cancel Selected.

Warning: Once a line item (product) or order (transaction) is canceled, it cannot be un-canceled.

  1. Once you fix the problem causing the exception, you must re-extract the transaction(s) from the Manage Integration Exceptions page. Re-extracting a transaction rebuilds and resubmits the transaction. If everything was corrected that caused the exception, the transaction(s) will process normally and will no longer be listed on the Manage Integration Exceptions page.
    • After you have taken steps to fix the exception, to re-extract (resubmit) multiple transactions from the Manage Integration Exceptions page, first enter the reason for the re-extraction in the Corrective Action Taken or Removal Reason field. You must provide the solution you took or the reason for this action. Then, select the check box(es) for the transaction(s) you want to resubmit and click Re-Extract Selected. You can select as many transactions as needed. To resubmit all transactions listed in the search results, click Re-Extract All.

    2. Tip: You can re-extract one or more transaction at the same time. If you fixed a problem that caused an exception on multiple transactions, you can select every transaction with the exception and re-extract. You do not have to re-extract each transaction separately.

    • Individual transactions may also be re-extracted (resubmitted) from the Transaction Details page (after you have taken steps to fix the exception). You must enter the reason for the action in the Corrective Action Taken or Removal Reason field before clicking Re-Extract.
