Creating Custom Bundle Offers

A custom bundle offer is one where the shopper can select from groups of products to create a bundle offer on your store.

Creating a custom bundle is very similar to creating other types of offers and many of the settings you can set for other offer types are the same for custom bundles. Because the process to create a custom bundle is very similar to other offers, this topic focuses on creating the groups for a custom bundle and entering the Display Content for the custom bundle.

Note: To learn how to create an offer, see Creating Marketing Offers. When you get to the third step (the Products and Pricing step) or the fourth step (the Display Content step) of the wizard, return to this topic for more detailed instructions.

Creating Groups in a Custom Bundle

Groups are created in the third step (the Products and Pricing step) of the offer wizard. An example of this step is shown below.

Setting the Currency

The Currency list at the top (in the Pricing Information area) is where you select the currency you want to be used for the offer. This is the currency that will be used to calculate the discount (if the offer provides one) and convert that discount into other currencies. We recommend you select the currency that is the default for the price list used by the products you will add to the offer.

Setting the Discount Type

The Discount Information area only appears if you chose "bundle" for the Discount Level setting in the previous step of the wizard. When discount an offer at the bundle level it means that the discount will apply to the entire bundle, rather than individual products in the bundle.

For the Discount Type option, select how you want to discount the bundle:

  • Select No Discount if you do not want to provide a discount with the offer. Custom bundle offers that do not provide a discount (and are "always triggered" can be used to require a shopper purchase multiple products together, rather than separately.
  • Select Amount Off if you want to discount the total cost of the bundle by a specific amount. The amount you enter in the Discount field will be deducted from the cost of the bundle using the currency you selected in the Pricing Information area above. For example, if you set the currency for the offer as USD (United States dollar) and the offer is redeemed by a shopper in EUR (euros), the discount amount will be calculated using the USD to EUR conversion rate, and that discount amount will then be deducted from the cost of the bundle.

  • Select Percent Off if you want to discount the total cost of the bundle by a percentage of the cost. The percentage you enter in the Discount field will be deducted from the total cost of the bundle. With the percent off discount type, the currency is not a factor (meaning that if you enter a percent off discount, the price will always be discounted by that percent regardless of currency).

Creating Groups

By default, when you get to the Products and Pricing step a group called "Primary Group" is already created and set as "mandatory." This is because you must have at least one mandatory group for the custom bundle. Once you define and save the first group, you can create additional groups and have them be mandatory or optional as needed.

An example of what you will see when you first enter this step is below.

  1. (Optional) In the Group Name field, enter a new name for the group. This name is only used in the Command Console to identify the group. In a later step (the Display Content step) you will enter a name for the group that shoppers will see on the store.
  2. If you want the shopper to have to buy more than one product from the group, in the Shopper Must Buy at Least field, enter the number of products you want the shopper to have to buy. 
    • The number you enter here will be the number of products that the shopper must buy because the first group is mandatory. When you are creating optional groups this minimum is only enforced if the shopper chooses products from the group.
    • This number cannot be higher than the total number of products you add to the group.
  3. In the Shopper Can Buy at Most field, enter the maximum number of products you want the shopper to be able to purchase (or have to purchase) from the group.
    1. This number cannot be higher than the total number of products you add to the group.
  4. In the Product Eligibility area, define how you want to define the products for the group:
    • Select Specific Products if you want to add products in different categories or you just want to add a few products from a single category.
    • Select Specific Categories if you want to add an entire category of products. Keep in mind there is a limit on the number of products you can add to group as defined in the custom bundle settings so if the categories you select have a total number of products more than this setting you will not be able to add those categories to the group.
  5. In the Products or Categories area (the name varies by the option you selected above), click Add. The Find Products or Find Categories window appears. Search for (and then select) the products or categories you want to add to the group. Keep in mind there is a limit on the number of products you can add to a group as define in the custom bundle settings. If you select more than this limit you will not be able to save the group.
  6. Once you have defined the group and added products to the group, click Save.
  7. Click Add Group and repeat the steps above to create additional groups for the offer. Keep in mind you must create at least two groups for custom bundle offers, and the number of groups you can create (and the number of products that are in each group) are determined by the custom bundle settings.

Entering the Display Content

The fourth step of the wizard is the Display Content step. This is where you enter the display names that the shopper will see for each of the groups you have created (as well as the "offer description in cart").

First, in the Offer Description in Cart field, enter the text that you want to appear in the cart when this offer is triggered. You do not have to enter a description if you want nothing to appear. Repeat this for any additional locales you have selected for the offer (click the locale tabs at top to switch between locales).

In the Storefront Display area, in the Display Name fields for each of the groups you have created, enter what you want to appear to shoppers when they are viewing the groups and products in the offer.

The name(s) you enter appear on the store with the group. In the example below you can see that we entered a "name" that actually helps direct the shopper on what to do. We recommend you enter a name that will inform your shoppers about the products in the group and any restrictions you have on the number of products they can select.

The Description you enter will not appear to shoppers and can be used to help others who see this offer later identify the contents of the group.
