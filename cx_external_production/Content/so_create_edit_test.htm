Creating Site Optimizer Tests

Site Optimizer tests are created using a wizard, which guides you through each step of the creation process. Once you create a test, it cannot be edited and must be ended or retired to stop running on your store. You can only edit Site Optimizer Tests with a status of Pending Start. This is a test with a Start Date that has not yet occurred.

Note: When a Site Optimizer Test is running, all offers of the same type as the offers in the Site Optimizer test are temporarily suspended. This is so the test can run without interference.

Before you can create a Site Optimizer Test you must first create and deploy the marketing offers you want to include in the test. The offers you create must have the offer trigger set as Site Test. Offers with this type of trigger do not become available on your site until you create a Site Optimizer Test that uses them. See Creating Marketing Offers for instructions on creating marketing offers.

When you create a test, if the Test Type you want is greyed out/unavailable or if you do not see any offers when you create the test cells, it means that there is not an associated marketing offer that can be used with the test. Keep in mind that only certain types of offers can be used with the specific test types. See Offer Types and Test Types to determine which Test Types are compatible with marketing offer types.

Previewing Tests

If needed, you can preview a test and see how each of the test cells will appear to a shopper. When you preview a test, you will see a version of your store that uses the first cell in the test. You can then click "Next" and "Previous" in the preview widget to move through the cells in the test.

To create or edit a Site Optimizer Test:

Role Needed: Site Optimizer1.) A feature in the Command Console that allows users to conduct site-wide A/B testing on a live site. 2.) A user role that provides access to Site Optimizer. Search this file for the "user role definitions" topic for more information.
  1. On the Marketing menu, select Site Optimizer Tests, and click create Site Optimizer Test. The Create Site Optimizer Test: Step 1 - Test Type page appears.
  2. On the Test Type page, select a Test Type and click Next. The type you select must be compatible with the types of offers you can have created for the test.

    If an option is grayed out and unavailable it means that you have not yet created (or deployed) a marketing offer associated with the test type. Once you create and deploy a marketing offer, the associated Site Optimizer Test type will become active and available on this page. Keep in mind that marketing offers that are deployed with an Offer Type of Site Test are not made live on your site until you create a Site Optimizer Test with them.
    • Content – Measures the effectiveness of a store home page banner point of promotion (POP) by testing three different creative designs. Compatible offer types: Discount, Feature Products
    • Product Discount – Measures specific products’ discounts. For example, you can test the sales performance of Product A at 5% Off Discount versus 10% Off and 25% Off. Compatible Offer Types: Banner, Cross-Sell, Up-Sell, Feature Products
    • Order Discount – Compares the impact of half of your shoppers receiving a “10% Off your entire order” promotion versus the other half receiving a “20% off your entire order”. Compatible Offer Types: Discount
    • Product Shipping Discount – Tests free ground shipping for Product A versus 50% overnight shipping for Product A to determine best sales per visit performance. Compatible Offer Types: Shipping Individual Product, Shipping Product Present, Shipping Product Present + Destination + Shipping Method
    • Order Shipping Discount – Tests free ground shipping for all orders versus 50% overnight shipping for all orders to determine best sales per visit performance. Compatible Offer Types: Shipping Order Value, Shipping Order Value + Shipping Method, Shipping Order Value + Destination
    • Cross-Sell/Up-Sell/Bundle – Tests the cross sell of Products 1,2 with Product A versus cross selling Products 3,4 with Product A to determine which children products have the best cross sell attachment rates. Compatible Offer Types: Bundle, Cross-Sell, Up-Sell
    • Content Layout – Tests two or more versions of markup in a content layout against each other, to see which performs better.
  1. On the Information page:
    1. In the Test Name field, enter a name for the test. This will be shown in the reporting results for the test.
    2. In the Description field, enter a description for the test.
    3. Enter a Start Date for the test. If this date is set in the future, the test will activate on your site on that date.
    4. Enter an End Date for the test. The end date is for data collection purposes only. Tests are not completed or retired until they are promoted, retired or ended manually.
    5. If the Product list appears, select the product you want to test. The products in the list are those that were set up with a marketing offer (which you will choose in the next step) with the trigger of Site Test.
    6. If the Point of Promotion list appears, select the POP you want to use with the test. The POPs in the list are those that were set up with a marketing offer (which you will choose in the next step) with the trigger of Site Test.
    7. Click Next to move on the next step of the wizard.
  2. On the Levels page, from the Offer list, select an offer for a test level and click Add. Each test level becomes a possible offer to be shown to a shopper. Repeat as often as needed to add all the levels you want to test.

You can select <no offer> from the list to add a test level that contains no offer. If your store has an existing marketing offer that would compete with the test it may appear to customers who are directed to the <no offer>  level.

If you do not see any offers in the list, or do not see the offer you expect, check the following:
  • Offers that do not have a trigger of Site Test will not appear. Go back to your marketing offers and check if this is set properly.
  • Offers must be deployed to be used in Site Optimizer Tests. When you deploy an offer with the trigger of Site Test, the offer will not appear on your live store until it is part of a deployed Site Optimizer Test.
  • The Test Type selected for the Site Optimizer Test filters the list of marketing offers that can be added to the test. See the information above to determine which Test Types are compatible with offer types.
  • Offers that are current in other test levels are not available for another test.
  1. On the Levels page, set up the levels as needed and click Next when done to move on to the next step:
    1. In the Control column, select the test level in the list that you want to be the Control group. Reporting will indicate that this test level is the control group to be compared with the other test levels. There can be only one control group per test.
    2. In the Split fields, enter a percentage for each level. The split determines the percentage of visitors that will see this offer and must add up to 100 across all test levels.

      The split is randomly assigned when the shopper first visits your storefront. The assigned test level is shown to the shopper during their entire session. Another test level is randomly assigned if the shopper completes an order or clears their browser cookies.
    3. To remove a level from the test, select the check box for the level and click Delete.
  2. On the Summary page, review the test you have created or edited.
    • If you are creating a new test, you can preview the test as it will appear on your site. Click Preview Test in the taskbar. A new window opens and displays the home page of the store with the first test level activated.

      To switch from test level to test level, click Next and Previous in the Site Optimizer preview widget in the upper right-hand corner of your browser window. This will activate each test level in the test and display it in the window.

    • When you are done previewing the test levels, click Finish to create the test. This will activate the test on your store and visitors will be randomly assigned to each of the test levels in the test according to the split defined.
    • To cancel creating the test, click Cancel.
  3. After you create the test you will want to view the test results, and promote test levels or end the test. See Viewing Test Results & Ending or Promoting Tests.
