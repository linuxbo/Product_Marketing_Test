Configuring Customer Service Settings

On the Configure Customer Service Settings page you can modify settings for your site that impact certain Customer Service features and options.

Note: Some of the settings discussed on this page will not be seen by all users. Only those settings that can be used on your site should appear, and some options appear and disappear based on whether other settings have been enabled.

To configure customer service settings:

Role Needed: Customer Service Configuration ManagerA user role that allows the user to configure customer service settings. Search this file for the "user role definitions" topic for more information.
  1. On the Administration menu, select Site, and click Configure Customer Service Settings. The Configure Customer Service Settings page appears.
  2. Make changes to the settings you want, using the information below as a guide. Don't forget to click Save when you are done.

Settings Definitions

  • Returns
  • Refunds

Returns

The Returns settings allow you to specify Reason Codes for returns (including canceling an RMA).

Reason Codes:

  • Select Use Digital River's Default Reason Codes if you want to use the standard set of reason codes Digital River created for all clients.
  • Select Use Customized Reason Codes to create your own reason codes, or choose which reasons codes are available for your store. To learn more about configuring (and creating) reason codes, see Configuring Reason Codes.

Refunds

The Refunds settings allow you to specify Reason Codes for refunds.

Reason Codes:

  • Select Use Digital River's Default Reason Codes if you want to use the standard set of reason codes Digital River created for all clients. Select Use Customized Reason Codes to create your own reason codes and/or choose which reason codes can be used by your different sites (if your company has multiple sites). To learn more about configuring (and creating) reason codes, see Configuring Reason Codes.
